# TELEMT Install

Автономный Bash-скрипт для установки и обслуживания `TELEMT`-прокси на `Ubuntu 24.04` с запуском в `Docker`.

Скрипт рассчитан на чистый VPS и старается не требовать ручной подготовки сервера перед запуском.

## Что делает скрипт

Скрипт `telemt-manager.sh`:

- проверяет, что система запущена на Ubuntu;
- при необходимости устанавливает базовые системные пакеты;
- при необходимости устанавливает Docker и Docker Compose из официального репозитория Docker;
- позволяет использовать домен или публичный IP сервера;
- проверяет домен:
  - синтаксис;
  - DNS resolve;
  - соответствие IP текущему VPS;
- предлагает выбрать внешний порт для прокси;
- создает конфигурацию `TELEMT`;
- загружает образ `ghcr.io/telemt/telemt:latest`;
- запускает контейнер `telemt`;
- проверяет успешный запуск контейнера и прослушивание порта;
- выводит готовую `tg://`-ссылку для Telegram;
- умеет показать ссылку повторно;
- умеет удалить установку полностью, кроме самого Docker.

## На каких компонентах работает

Основные компоненты:

- `Ubuntu 24.04`
- `Docker`
- `Docker Compose`
- `TELEMT`

Системные утилиты, которые использует скрипт:

- `apt-get`
- `curl`
- `openssl`
- `getent`
- `ss`
- `gpg`
- `dpkg`
- `systemctl`

Docker-компоненты:

- `docker-ce`
- `docker-ce-cli`

- `ghcr.io/telemt/telemt:latest`

## Возможности меню

После запуска скрипт показывает меню:

1. `Установить / Обновить`
2. `Удалить`
3. `Показать ссылку`
4. `Статус`
0. `Выход`

## Что происходит при установке

Сценарий установки:

1. Проверка ОС и прав доступа.
2. Установка недостающих базовых пакетов.
3. Установка Docker и Compose, если они отсутствуют.
4. Выбор режима:
   - использовать домен;
   - использовать публичный IP.
5. Проверка домена или определение IP.
6. Выбор внешнего порта.
7. Генерация нового секрета или повторное использование существующего.
8. Создание рабочих файлов.
9. Загрузка и запуск контейнера.
10. Вывод готовой ссылки для Telegram.

## Что создается на сервере

Рабочая директория:

```text
/opt/telemt
```

Внутри создаются:

- `config.toml` — конфигурация `TELEMT`
- `docker-compose.yml` — compose-файл
- `telemt.env` — сохраненные параметры:
  - домен или IP;
  - внешний порт;
  - секрет доступа

## Что удаляет скрипт

При выборе `Удалить` скрипт удаляет:

- контейнер `telemt`
- compose-ресурсы `TELEMT`
- образ `ghcr.io/telemt/telemt:latest`
- рабочую папку `/opt/telemt`

Скрипт не удаляет:

- Docker
- Docker Compose
- другие контейнеры
- другие образы
- другие volumes, не относящиеся к `TELEMT`

## Проверки и защита от ошибок

В скрипте есть:

- проверка ОС;
- проверка прав доступа;
- проверка пользовательского ввода;
- проверка домена;
- проверка занятости порта;
- проверка Docker;
- проверка запуска контейнера;
- проверка того, что выбранный порт реально слушается.

Если на каком-либо этапе возникает ошибка, скрипт:

- показывает понятное сообщение;
- подсказывает, что нужно исправить;
- возвращает пользователя в главное меню.

На интерактивных шагах можно нажать `*`, чтобы вернуться в главное меню.

## Быстрый запуск

Локально:

```bash
bash -n telemt-manager.sh && sudo bash telemt-manager.sh
```

Запуск одной командой на вашем VPS:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/filosoff-dev/telemt-install/main/telemt-manager.sh)
```

## Что получает пользователь в результате

После успешной установки скрипт:

- показывает, что `TELEMT` работает;
- выводит готовую ссылку для Telegram;
- показывает инструкцию:
  - скопировать ссылку;
  - открыть ее через приложение Telegram;
  - после этого прокси запустится автоматически.

После успешного результата скрипт завершает работу и возвращает в командную строку.

## Ограничения

- основной целевой сценарий — `Ubuntu 24.04`;
- для установки нужен доступ в интернет;
- для установки нужен `sudo` или запуск от `root`;
- если используется домен, его DNS-записи должны уже указывать на текущий VPS;
- скрипт ориентирован на запуск `TELEMT` на портах, подходящих под сценарий Cloudflare.
